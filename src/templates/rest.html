Lost Service Calls

  • lost_key
    • URL: http://127.0.0.1:8080/rest/lost_key

    • Description:
      Makes a request to return the LOST public key.
      Returns key and result OK if passed, otherwise fails

    • Usage:
      Request takes no arguments.

  • activate_user
    • URL: http://127.0.0.1:8080/rest/activate_user

    • Description:
      Reactivates LOST access for a user or generates a new user account if needed.
      Returns result OK if passed, otherwise fails.

    • Usage:
      Call is followed by username in which to activate/create
      Example( http://127.0.0.1:8080/rest/activae_user larry)

  • suspend_user
    • URL: http://127.0.0.1:8080/rest/suspend_user

    • Description:
      Revokes access for a user. If the user does not exist or access has already been revoked, this call has no effect.
      Returns result OK if passed, otherwise fails.

    • Usage:
      Call is followed by username in which to revoke.
      Example( http://127.0.0.1:8080/rest/suspend_user larry)

  • list_products
    • URL: http://127.0.0.1:8080/rest/list_products

    • Description:
      Requests a listing of all products in LOST based on a filter criteria. Additional filters may be provided and additional data may be returned. Asset information may not be provided by this call.
      Returns product information.

    • Usage:
      Call is followed by description of the product.
      Example( http://127.0.0.1:8080/rest/list_products notepad)

  • add_products
    • URL: http://127.0.0.1:8080/rest/add_products

    • Description:
      Adds products to LOST. Either all products are added during this call or none are. Duplicate products cause the request to fail. Returns OK if passed, otherwise fails.

    • Usage:
      Call is followed by a json list describing the new products.
      Example( http://127.0.0.1:8080/rest/add_products "new_producsts": { {"vendor: "Dunder Mifflin", "description": "LOST letter size notepad", "alt_description": "Children's storybook", "compartments": ["adm:s"] } }

  • add_asset
    • URL: http;//127.0.0.1:8080/rest/add_asset

    • Description:
      Adds a new asset to LOST. Returns OK if passed otherwise fails.

    • Usage:
      Call is followed by a json list describing the new asset.
      Example( http://127.0.0.1:8080/rest/add_asset { "vendor": "Dunder Mifflin", "description": "LOST letter size notepad", "compartments": ["wpn:ts"], "facility": "HQ"}
